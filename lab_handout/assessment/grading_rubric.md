# Lightweight Pass/Fail Rubric

This lab is graded pass/fail. Topic 7 PWM is part of the required lab flow. Topic 8 is optional and ungraded.

| Category | Pass evidence |
| --- | --- |
| Vivado project navigation | Student can identify Design Sources, Simulation Sources, and Constraints. |
| Simulation | Student ran at least one behavioral simulation and can explain one observed waveform behavior. |
| RTL understanding | Student distinguishes combinational logic, counter-based sequential logic, and FSM state logic. |
| FPGA implementation | Student completed the AC701 implementation successfully. |
| Hardware target understanding | Student can explain why AC701 is used for implementation reports and why AUP-ZU3 demos use prebuilt bitstreams. |
| Timing report | Student recorded WNS/TNS and can state whether timing is met. |
| Critical path | Student recorded a startpoint and endpoint and can describe the path in simple terms. |
| Power report | Student recorded power values and understands static versus dynamic power, estimate assumptions, and the external-load limitation. |
| PWM | Student completed the PWM TODOs, ran the PWM simulation, showed `PWM_STUDENT_TEST_PASS` to the responsible TA, and related duty cycle to LED brightness. |
| Optional rainbow demo | No report credit required; students may program the prebuilt rainbow demo if they finish early. |
| Conceptual answers | Student answered the pass/fail questions with reasonable technical understanding. |

Suggested pass threshold: all major evidence categories attempted, with no serious misconception such as treating `tick_1s` as a new clock or claiming Vivado power is a physical measurement from the board.
